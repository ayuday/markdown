# 列表语法速查

## 无序列表

```markdown
- 项目 1
- 项目 2
- 项目 3

* 项目 1
* 项目 2
* 项目 3

+ 项目 1
+ 项目 2  
+ 项目 3
```

## 有序列表

```markdown
1. 第一项
2. 第二项
3. 第三项
```

## 嵌套列表

```markdown
- 一级项目
  - 二级项目
    - 三级项目
      - 四级项目

1. 有序项目 1
   - 无序子项目
   - 无序子项目
2. 有序项目 2
   1. 有序子项目
   2. 有序子项目
```

## 任务列表

```markdown
- [x] 已完成任务
- [ ] 未完成任务
- [x] 另一个已完成任务
```

## 列表中的其他元素

### 段落

```markdown
1. 第一项

   这是第一项的第二段落。

2. 第二项
```

### 代码块

```markdown
1. 安装依赖

   ```bash
   npm install
   ```

2. 启动项目
```

### 引用

```markdown
1. 重要提示

   > 请注意安全

2. 下一步
```

## 最佳实践

| 建议 | 说明 |
|------|------|
| ✅ 保持符号一致 | 整个文档使用同一种符号 |
| ✅ 正确缩进 | 子列表使用 2-4 个空格缩进 |
| ✅ 避免过深嵌套 | 最多 3-4 层嵌套 |
| ❌ 不要跳级 | 避免从一级直接到三级 |

## HTML 输出

| Markdown | HTML |
|----------|------|
| `- 项目` | `<ul><li>项目</li></ul>` |
| `1. 项目` | `<ol><li>项目</li></ol>` |
| `- [ ] 任务` | `<input type="checkbox">` |
