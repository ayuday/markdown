# Markdown 标题语法

标题是组织文档结构最重要的元素。Markdown 支持两种标题语法：ATX 风格和 Setext 风格。

## ATX 风格标题

要创建标题，请在单词或短语前面添加井号 (`#`)。井号的数量代表了标题的级别。

```markdown
# 一级标题
## 二级标题  
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### 渲染效果

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

### 可选的关闭语法

你可以选择性地「关闭」ATX 风格的标题，这纯粹只是美观用的：

```markdown
# 一级标题 #
## 二级标题 ##
### 三级标题 ###
```

行尾的 `#` 数量不需要和开头一样，行首的井字符数量决定标题的阶数。

## Setext 风格标题

还可以在文本下方添加任意数量的 `=` 号来标识一级标题，或者 `-` 号来标识二级标题。

```markdown
一级标题
========

二级标题
--------
```

### 渲染效果

一级标题
========

二级标题
--------

## 标题最佳实践

### 兼容性考虑

当井号（`#`）和标题文本之间没有空格时，各 Markdown 应用程序的处理方式是不一样的。为了兼容考虑，请在井号和标题文本之间添加一个空格。

| ✅ 推荐做法 | ❌ 不推荐 |
|----------|---------|
| `# 这是标题` | `#这是标题` |

### 空行的使用

为了兼容性，请在标题前后都添加空行。

```markdown
这是一段文本。

# 这是标题

这是另一段文本。
```

### 标题层级

- 每个文档建议只使用一个一级标题
- 保持标题层级的逻辑性，不要跳级
- 标题应该反映文档的结构层次

### 标题文本规范

- 标题文本简洁明确
- 避免使用标点符号（特别是句号）
- 使用动名词或名词短语
- 保持同级标题格式一致

## HTML 输出

Markdown 标题会被转换为相应的 HTML 标题标签：

```markdown
# 标题       →  <h1>标题</h1>
## 标题      →  <h2>标题</h2>
### 标题     →  <h3>标题</h3>
#### 标题    →  <h4>标题</h4>
##### 标题   →  <h5>标题</h5>
###### 标题  →  <h6>标题</h6>
```

## 标题 ID（扩展功能）

许多 Markdown 处理器会自动为标题生成 ID，用于创建锚点链接：

```html
<h1 id="markdown-标题语法">Markdown 标题语法</h1>
```

这样你就可以链接到特定的标题：

```markdown
[链接到标题](#markdown-标题语法)
```

## 实际应用示例

以下是一个展示良好标题结构的文档示例：

```markdown
# 产品使用手册

## 快速开始

### 安装步骤

#### 系统要求

#### 下载和安装

### 基本配置

## 高级功能

### 自定义设置

### 插件管理

## 故障排除

### 常见问题

### 联系支持
```

这种层次分明的标题结构使文档易于阅读和导航。
